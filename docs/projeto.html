Brasileirão Fantasy

Liga fantasy privada do Brasileirão pra um grupo de 9 amigos. Cada um tem um time fixo (~26 jogadores) e pontua semanalmente. Tem mercado interno (trocas + free agents), draft com prioridades, e rodada ao vivo com substituições automáticas.

Deno Fresh Deno KV TypeScript + Preact jsDelivr CDN Cartola API Google OAuth

Stack

Frontend

  • Fresh (framework SSR-first do Deno) — server-rendered HTML com ilhas Preact pra interatividade pontual
  • Preact nas islands (formulários, modais, swap de jogador, mercado, etc)
  • CSS custom em static/bf-styles.css — sem framework
  • Design system próprio (dark mono, JetBrains Mono + IBM Plex Sans, accent lime)

Backend

  • Deno Deploy (edge runtime serverless) hospedando o Fresh
  • Deno KV pra estado persistente (elencos, mercado, sessions, draft, simulação)
  • Cartola API (pública) pra catálogo de atletas, pontuação ao vivo e partidas
  • Google OAuth pra SSO + login admin via senha
  • Crons do Deno: sync diário do catálogo + atualização de 5min na rodada

Imagens e CDN

  • Imagens (cutouts de jogadores, escudos, logos) em repo público separado Iuri07/brasileirao-fantasy-assets
  • Servidas via jsDelivr (cache global multi-PoP)
  • Helper lib/cdn.ts reescreve URLs em prod; em dev usa symlink local

Geração de assets

  • ogol.com.br scraping pra fotos dos jogadores
  • rembg (Python CLI) pra remover fundo das fotos (gera cutout PNG transparente)
  • Roda local (rembg não funciona em Deno Deploy); resultado é committado no repo de assets

Por que essa stack?

  • Deno Fresh + KV: SSR completo (rápido pra cargas leves), KV embutido (sem provisionar DB separado), deploy global em 1 click. Ideal pra app de 9 usuários.
  • Cartola API pública: já tem todo o catálogo de jogadores brasileiros, pontuação oficial, partidas. Tirou da equação ter que manter esses dados.
  • jsDelivr CDN: imagens são as request mais numerosas; Deno Deploy edge soma TTFB acumulado. Off-load pra CDN específico = páginas 14× mais rápidas em assets.

["historico", chave] { rodada → pontos } por elenco Cron atualizar (após rodada)

Performance e otimizações

Resposta SSR

  • KV reads em Promise.all agressivo (1 round-trip por block)
  • Cartola /atletas/mercado chunked em KV (TTL 5min, fits no limite de 64KB/value via chunks de ~200 atletas)
  • getInteressadosBatch via kv.list (1 trip) em vez de N gets
  • Server-Timing em routes/* pra rastrear regressões

Browser

  • Cache-Control: max-age=1y, immutable em CSS/JS/imagens (URLs versionadas via ?v=N)
  • <link rel="prefetch"> ao hover/touch de links internos
  • <link rel="preconnect"> pro CDN do jsDelivr (TLS antecipado)
  • Fade-out de navegação CSS-only (sem setTimeout)

Imagens

  • jsDelivr servindo do repo Iuri07/brasileirao-fantasy-assets (público)
  • Helper lib/cdn.ts reescreve URLs em prod; dev usa symlink local
  • TTFB médio: ~45ms (vs ~600ms do Deno Deploy serve estático)

Build / repo

  • Imagens fora do repo de código (~72MB)
  • Manifests TS auto-gerados (lib/cutouts-manifest.ts, lib/players-manifest.ts) pra evitar Deno.readDirSync em runtime
  • Cache in-process do client Cartola por path (TTL específico)
